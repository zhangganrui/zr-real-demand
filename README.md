# 真需求 · real-demand

> A Claude Code / Claude Agent **Skill** that interrogates a fuzzy idea until you can tell whether it's a *real demand* or a self-deceiving fake one — before you waste a single day designing or building it.
>
> 一个用于 Claude Code / Claude 的 **Skill**：在你动手设计、写代码之前，先把一个模糊的想法逼问到"真假可辨"，判定它是**真需求**，还是自我感动的**伪需求**。

基于梁宁《真需求》(2024) 的「**价值 - 共识 - 模式**」闭环，融合其《产品思维30讲》的需求挖掘工具（四情绪 / 痛痒爽 / 大明笨笨小闲）。

---

## 它解决什么问题

绝大多数产品不是死于做得不好，而是死于**一开始就在做一个伪需求**——一个"我觉得很酷""大家应该会需要"的东西。等到设计、开发都投进去了，才发现没人愿意为它付出任何代价。

`real-demand` 就站在所有动手动作的**最前面（第 0 步）**，陪你把想法逼问清楚：

- **该不该做** —— 这是真需求，还是你的自我感动？
- **为谁做** —— 具体的人、具体的处境、具体的触发时刻（"所有人"是禁词）。
- **做什么价值** —— 功能 / 情绪 / 资产价值，用户认不认。

判定为伪需求，就到此为止——**省下后面所有设计和开发的成本，这正是它最大的价值。**

## 核心方法：真需求 →「价值 - 共识 - 模式」

```
                    真需求（底座 · 诚实面对真实处境）
                          │
        ┌─────────────────┼─────────────────┐
      价值               共识                模式
   为什么值得         别人认不认          怎么持续活下来
  功能+情绪+资产    用户/市场/社会共识    能力系统/变现/分配
```

诊断按 **先挖、再判、后决** 三段走：

1. **A 段 · 挖**（手术刀）：情绪定位 → 痛/爽/痒 → 大明/笨笨/小闲 → 代价测试 / 替代方案 / 消失测试。
2. **B 段 · 判**（闭环）：价值三角 → 价值共识 → 模式是否成立。
3. **C 段 · 决**：真需求判决书（做 / 不做 / 改）+ 最小验证实验。

> 核心信念：**用户嘴上说的不是真需求。真需求是用户愿意为之付出代价的东西——钱、时间、改变习惯、暴露隐私、承担风险。不愿付任何代价的，都是伪需求。**

## 安装

把 skill 目录拷进你的 Claude skills 目录即可：

```bash
git clone https://github.com/zhangganrui/real-demand.git
cp -r real-demand/real-demand ~/.claude/skills/
```

重启 Claude Code 后，说"我有个想法 / 帮我看看这个需求 / 这是不是真需求"即可触发。

## 用法 / 触发

直接把你的想法丢给它，例如：

- "我想做个 XXX，帮我看看是不是真需求"
- "这个功能值不值得做？"
- "帮我把这个想法理清楚"

它会**一次只问你一个最戳真伪的问题**，逐步把想法逼清，最后产出一份**真需求诊断报告**（含 10 项快速打分）。

## 一个真实例子

作者用它诊断过一个"AI 知识看板"的想法。它没有顺着"做个看板"往下设计，而是用**代价测试 + 消失测试**把原始解法证伪了——发现用户根本不会回看收藏夹，真需求其实是"**把'我必须主动回看'这个负担彻底拿掉**"。原始想法被改了方向，省下了一整套白做的设计。

这就是这个 skill 该有的样子：**不是夸你想法好，而是在你花钱花时间之前，帮你发现它是个伪需求。**
